🔑 핵심 개념: 하네스 (Harness)

"에이전트가 7명인 건 알겠는데, 어떻게 이 에이전트들이 제 의도대로 움직이는 걸까?"

답은 하네스입니다. 하네스는 에이전트가 움직이는 규칙과 틀입니다. 각 에이전트의 역할 정의, 입력/출력 형식, 해야 할 것과 하지 말아야 할 것의 목록, 승인 게이트, 캐릭터 가이드가 모두 하네스에 정의됩니다.

세팅 시간: 캐릭터 가이드 정리에만 며칠이 걸렸지만, 이것은 딱 한 번의 투자입니다. 한 번 완성되면 이후 반복 실행은 30분입니다.

👥 7명의 에이전트 상세

1️⃣ 리서처

역할: YouTube, 뉴스, 블로그, SNS를 병렬로 검색합니다.

핵심 원칙: 출처가 없는 정보는 사용 금지. 검증되지 않은 통계를 팩트처럼 말하지 않음.

출력: 소스 MD 파일 (핵심 인사이트, 수치 데이터, 인용 가능한 발언, 비유 소재, 시각화 아이디어)

2️⃣ 스크림라이터

역할: 대본 작성 (Opus 모델 사용)

입력: 리서처 소스 파일 + 캐릭터 가이드 (톤, 화법, 규칙 8가지/금지 11가지) + 타겟 러닝 타임

언어 번역 예시: "API" → "주문 전달 경로", "배포" → "매장 오픈"

출력: 씬 단위 대본 (나레이션, 예상 시간, 시각 지시)

3️⃣ 서브타이틀 엔지니어

역할: SRT 자막 파일 생성

계산: 분당 509자 기준, 최대 50자, 블록 간 0.4초, 씬 경계 0.8초 갭

4️⃣ 보이스 엔지니어

역할: 텍스트를 음성으로 변환 (로컬 TTS 서비스)

특징: 김효일 목소리 샘플로 클론 보이스 제작. 실제 오디오 길이 기준으로 자막 타이밍 리스케일

5️⃣ 씬 디자이너 ⭐ 핵심

역할: 각 씬의 시각적 디자인 (Opus 모델)

혁신: Remotion을 사용하여 React 컴포넌트로 영상을 만듭니다. 배경색, 텍스트 크기, 카드 레이아웃, 애니메이션이 모두 TypeScript로 정의됩니다. AI가 코드를 잘 쓰기 때문에 "레이어 케이크 다이어그램을 넣어줘"라고 요청하면 자동으로 생성합니다.

디자인 규칙: style.json 파일에 색상 팔레트, 폰트 스케일, 여백 토큰, 카드 스타일, 모션 시그니처 정의

6️⃣ 렌더러

역할: React 컴포넌트를 Remotion 프로젝트에 통합하고 최종 영상 렌더링

소요 시간: 10~12분 (16:9, YouTube 업로드 가능 해상도)

7️⃣ QA 리뷰어

역할: 최종 품질 검수 (Opus 모델)

체크리스트 26개:
  • 포지셔닝 4항목 (니치 소유, 실제 업무 수행 등)
  • 콘텐츠 구조 5항목 (결과물 표시, 단일 주제 등)
  • 실루엣 4항목 (경험 드러내기, 출처 명시)
  • 편집과 비주얼 4항목 (3~5초 시각 변화, 빈 화면 0% 등)

필수 7개 미통과 시: 문제 단계에서 자동으로 리셋하고 재실행

🎯 본질과 응용

파이프라인의 본질은 명확합니다: 큰 작업을 7단계로 나누고, 각 단계에 전문 에이전트를 배치하고, 입력/출력을 표준화한 것입니다.

이것은 기획자가 하던 일과 같습니다. 브리프를 쓰고 팀원에게 넘기고 결과를 확인하고 피드백을 주고. 다만 이 팀원이 AI이고 응답이 즉각이라는 것뿐입니다.

다양한 직종에 적용:
  • 마케터: 콘텐츠 기획 → 배포
  • 운영팀: 데이터 수집 → 주간 리포트
  • 인사: 채용 공고 작성 → 후보자 검토
