🎥 YouTube STT 분석 파이프라인 구현 가이드

이 문서는 YouTube 영상을 자동으로 수집하여 텍스트로 변환(STT)하고, AI를 통해 요약하여 사용자에게 전달하는 시스템의 구현 방법을 설명합니다. 다른 에이전트가 이 기능을 구현할 때 이 가이드를 따라 환경을 구축하고 스크립트를 구성하십시오.

🏗️ 1. 시스템 아키텍처 (Data Flow)

YouTube Data API → yt-dlp (오디오 추출) → Faster-Whisper (STT 변환) → LLM (내용 요약) → 결과 저장 및 전송

상세 단계:

  • 수집 (Collection): YouTube API를 통해 특정 채널의 최신 영상 URL 리스트를 확보합니다.
  • 추출 (Extraction): yt-dlp를 사용하여 영상에서 오디오 스트림만 고품질로 다운로드합니다.
  • 변환 (Transcription): Faster-Whisper (large-v3 모델)를 사용하여 오디오를 텍스트로 변환합니다. (GPU 가속 필수)
  • 요약 (Summarization): 변환된 텍스트를 LLM(Claude 등)에 전달하여 사용자가 읽기 좋은 형태로 요약합니다.
  • 전송 (Delivery): 요약 결과를 파일로 저장하고, 텔레그램 등의 메시징 채널로 전송합니다.

🛠️ 2. 환경 설정 (Prerequisites)

📦 필수 패키지 및 도구

  • Python venv: 종속성 충돌 방지를 위해 전용 가상환경을 사용합니다.
  • FFmpeg: 오디오 처리 및 추출을 위한 필수 시스템 도구입니다.
    sudo apt install ffmpeg
  • Python 라이브러리:
    • yt-dlp: YouTube 영상/오디오 다운로드용
    • faster-whisper: 고성능 STT 엔진
    • torch: GPU 가속을 위한 딥러닝 프레임워크
    • transformers: 모델 로드 및 처리
🚀 GPU 최적화 (NVIDIA)
CUDA Toolkit 설치 필수. Faster-Whisper 실행 시 device="cuda" 및 compute_type="float16" 설정을 통해 속도를 극대화합니다.

📜 3. 핵심 스크립트 구조

A. youtube-stt-runner.py (개별 영상 처리)

단일 URL을 입력받아 STT 및 요약까지 완료하는 핵심 엔진입니다.

  • 입력: YouTube URL
  • 로직:
    1. yt-dlp로 .wav 또는 .mp3 추출.
    2. Faster-Whisper로 텍스트 변환 (긴 영상의 경우 5분 단위 청크 처리 권장).
    3. LLM API를 호출하여 요약본 생성.
    4. youtube_results/[영상제목]/ 폴더에 stt.txt와 summary.txt 저장.

B. youtube-daily-analysis.js (자동화 제어)

정기적으로 실행되어 채널을 모니터링하고 러너를 호출하는 오케스트레이터입니다.

  • 입력: API Key, Channel ID
  • 로직:
    1. youtube/v3/search 엔드포인트로 최신 영상 5개 추출.
    2. 반복문을 통해 각 영상 URL을 youtube-stt-runner.py에 전달.
    3. 모든 작업 완료 후 상태 보고.

⚠️ 5. 주의 및 최적화 팁

1. API 할당량: YouTube Search API는 할당량 소모가 큽니다. 하루 1~2회 실행을 권장합니다.
